# 项目恢复指南（中断后快速上手）

> **目标**: 项目中断后，30 秒内恢复上下文，继续开发  
> **适用**: AI 驱动开发的项目恢复

---

## 🎯 核心理念

### 问题
- 项目开发中途被打断
- 第二天继续开发，忘记进度
- 多人协作，不知道当前状态
- AI 需要重新理解项目

### 解决方案
**项目状态文件 + AI 恢复 Prompt**

## 🚀 快速恢复流程

### Step 1: 查看项目状态（30 秒）

```bash
# 进入项目目录
cd /path/to/project

# 查看项目状态
cat PROJECT-STATUS.md

# 查看最近提交
git log -5 --oneline

# 查看当前分支
git branch
```

### Step 2: 发送恢复 Prompt 给 AI（30 秒）

```
# 项目恢复指令

我需要继续开发这个项目，请帮我快速恢复上下文。

## 项目状态

[粘贴 PROJECT-STATUS.md 的内容]

## 项目配置

[粘贴 PROJECT-BASE-INFO.md 的内容]

## 请执行以下操作

1. 阅读项目状态，确认当前进度
2. 查看"进行中模块"，了解当前任务
3. 查看"已知问题"，了解需要解决的问题
4. 查看"下一步计划"，了解接下来要做什么
5. 告诉我：
   - 当前正在做什么
   - 下一步应该做什么
   - 需要注意什么问题

开始恢复！
```

## 📝 状态更新规范

### 何时更新 PROJECT-STATUS.md

1. **完成一个模块**: 移动到"已完成模块"
2. **开始新任务**: 添加到"进行中模块"
3. **发现问题**: 添加到"已知问题"
4. **做出决策**: 记录到"关键决策记录"
5. **每天结束**: 更新"最后更新"时间

### 更新方式

```bash
# 手动更新
vim PROJECT-STATUS.md

# 或者让 AI 更新
"请更新 PROJECT-STATUS.md，标记登录接口已完成"
```

| 方式 | 恢复时间 | 准确度 | 说明 |
